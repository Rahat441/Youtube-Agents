Project Overview

The long-term pipeline is designed to move from raw topic to full YouTube video project: 

ResearchAgent
  -> StrategyAgent
  -> IdeaAgent
  -> OutlineAgent
  -> ScriptAgent
  -> CritiqueAgent
  -> PackageAgent

ResearchAgent

Collects and organizes YouTube evidence. It does not choose the final lane, angle, idea, or script.

StrategyAgent

Reads research evidence, scores niche lanes, chooses a recommended lane and angle, and writes strategy.

IdeaAgent

Reads research and strategy artifacts, generates scored candidate ideas, and selects the top idea.

Design rule: each agent should do one job and write a structured artifact. This keeps the pipeline inspectable and makes it easier to improve one stage without rewriting the whole system.

Current Workflow

Run ResearchAgent, StrategyAgent, Then IdeaAgent

Step 1: collect YouTube evidence.

python3 main.py research \
  --topic "walnuts" \
  --audience "health curious adults" \
  --style "documentary explainer" \
  --topic-context "nutrition health benefits eating walnuts science" \
  --content-type scriptable \
  --language english \
  --source youtube \
  --youtube-max-results 10 \
  --youtube-max-queries 10 \
  --youtube-comments-per-video 5

Step 2: run strategy from the generated research.json.

python3 main.py strategy \
  --research workspace/agent_runs/YYYYMMDD_HHMMSS_topic/research.json

Step 3: generate ideas from the research and strategy artifacts.

python3 main.py ideas \
  --research workspace/agent_runs/YYYYMMDD_HHMMSS_topic/research.json \
  --strategy workspace/agent_runs/YYYYMMDD_HHMMSS_topic/strategy.json \
  --ideas-per-run 5 \
  --provider template

Outputs are saved in the same run folder:

workspace/agent_runs/YYYYMMDD_HHMMSS_topic/
  research.json
  strategy.json
  ideas.json

ResearchAgent

ResearchAgent is the evidence collector. It builds YouTube queries, calls the YouTube Data API, filters obvious bad matches, scores video performance, buckets competitor fit, extracts light patterns, and writes research.json.

Responsibility
  • Collect clean YouTube/topic evidence.
  • Generate neutral discovery queries from templates.
  • Collect video, channel, metric, duration, tag, and comment evidence when available.
  • Filter by source, content type, language, manual keywords, and basic topic relevance.
  • Score performance signals such as views_per_day, engagement_rate, outlier_ratio, and is_breakout.
  • Bucket competitor fit without deleting creative outliers.

ResearchAgent must not decide: final niche lane, final angle, script premise, title, outline, or package.

How Query Planning Works

ResearchAgent uses deterministic templates instead of AI-generated searches. For walnuts, it can build:

  • walnuts
  • walnuts explained
  • walnuts benefits
  • walnuts risks
  • walnuts mistakes
  • walnuts myths
  • walnuts what happens
  • walnuts science
  • walnuts beginner guide
  • walnuts side effects
  • what happens when you eat walnuts
  • walnuts nutrition facts

It can also append audience, style, manual context, and include keywords. It avoids forced lane searches like POV life as {topic}.

How YouTube Evidence Is Collected
  1. search.list finds videos for each query.
  2. videos.list retrieves title, description, tags, duration, statistics, and status fields.
  3. channels.list retrieves channel and subscriber data.
  4. commentThreads.list samples comments when comments are available.

Search uses relevance and view-count ordering for most queries. The main query also gets a date-order search to capture freshness.

How Filtering And Scoring Works

ResearchAgent filters hard rejects first:

  • manual exclude keyword matched
  • manual include keywords provided but missing
  • duration does not match content_type
  • language=english and title appears mostly non-English/non-Latin
  • made-for-kids when avoid_kids_content is true
  • basic topic relevance below threshold

Then it calculates performance fields:

  • views_per_day
  • engagement_rate
  • outlier_ratio = views / subscriber_count
  • is_breakout when velocity or outlier ratio is strong
Competitor Fit Buckets

Competitor fit is a soft scoring layer. It does not erase weird high performers by default. It labels them so StrategyAgent can make cleaner decisions while IdeaAgent can still inspect unusual inspiration later.

BucketPurpose
strong_fitDirect competitors matching topic, context, and format. Used first for strategy scoring.
adjacent_opportunityNearby videos that may inspire hooks, packaging, or sub-angles.
off_context_outlierHigh-performing but clearly different-context videos. Useful for creative awareness, not core strategy.
weak_fitKept evidence with weak fit. Usually not important for strategy.

StrategyAgent

StrategyAgent reads research.json, interprets the evidence, scores possible niche lanes, chooses a recommended lane and angle, and writes strategy.json.

Responsibility
  • Read ResearchAgent output.
  • Prefer strong_fit_competitors for scoring when available.
  • Score niche lanes from query matches, title patterns, breakout videos, audience fit, claim risk, and saturation.
  • Choose a recommended niche lane and secondary niche lane.
  • Choose a recommended angle, angle type, positioning, confidence, and key evidence.

StrategyAgent must not decide: final title, full outline, script, thumbnail package, or final video package.

How Lane Scoring Works

Each lane receives component scores:

ComponentMeaning
query_scoreWhether videos matched query purposes relevant to the lane.
title_pattern_scoreWhether common title patterns fit the lane.
breakout_scoreWhether breakout videos use lane-relevant keywords.
audience_fit_scoreWhether audience, style, and manual context fit the lane.
risk_fit_scoreWhether the lane responsibly fits risky claims such as health or finance.
saturation_penaltyPenalty when a lane appears common enough to require differentiation.

If strong_fit_competitors exists, StrategyAgent uses that bucket for core scoring. If not, it falls back to competitor_videos for backward compatibility.

IdeaAgent

IdeaAgent reads research.json and strategy.json, generates template-based or LLM-assisted candidate video ideas, scores them, selects the highest-scoring idea, and writes ideas.json. The LLM only generates raw candidates; IdeaAgent still normalizes, scores, sorts, and writes the stable artifact.

Responsibility
  • Use StrategyAgent's recommended lane, angle type, positioning, confidence, and key evidence.
  • Use ResearchAgent's strong-fit competitor evidence and claim risks.
  • Generate candidate video ideas.
  • Score ideas for strategy fit, evidence support, click potential, differentiation, feasibility, audience fit, and risk.
  • Select the highest-scoring idea for OutlineAgent.

IdeaAgent must not decide: full outline, script, thumbnail package, or final edited title.

Current Limitations

Research Limitations
  • The YouTube Data API can return odd matches for ambiguous topics.
  • Competitor-fit scoring is heuristic and may mis-bucket some videos.
  • Comment collection can fail when comments are disabled or quota/API permissions prevent access.
  • The language filter is a simple non-Latin ratio check, not full language detection.
  • Offline mode writes a valid artifact shell but does not invent stats.
  • No API response cache exists yet, so repeated research can spend quota.
Strategy Limitations
  • Lane scoring is deterministic and heuristic, not an LLM reasoning pass.
  • Saturation penalty is still simple.
  • StrategyAgent can only score the lanes currently encoded in the code.
  • StrategyAgent does not yet write a human-readable markdown report.
  • It does not validate JSON schemas at runtime.

Future Enhancements To Consider

Near-Term

  • Add a command that runs ResearchAgent, StrategyAgent, and IdeaAgent in one workflow.
