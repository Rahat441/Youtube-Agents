YouTube Agents Rebuild Plan

Goal
Build a local-first Python pipeline that creates YouTube video projects from a topic, one agent at a time.

Target pipeline
1. ResearchAgent
2. StrategyAgent
3. IdeaAgent
4. OutlineAgent
5. ScriptAgent
6. CritiqueAgent
7. PackageAgent

Current scope
ResearchAgent v1, StrategyAgent v1, and IdeaAgent v1.

ResearchAgent responsibility
Collect clean topic and YouTube evidence. It must not decide the final niche lane, recommended angle, premise, or script direction.

ResearchAgent inputs
- topic
- audience
- style
- manual_brief
- youtube_settings

ResearchAgent outputs
- research.json
- query_plan
- evidence_status
- market_stats
- competitor_videos
- strong_fit_competitors
- adjacent_opportunity_videos
- off_context_outliers
- breakout_videos
- title_patterns
- viewer_pains
- claims_to_verify
- source_errors
- handoff_contract

ResearchAgent CLI
python3 main.py research --topic "walnuts" --audience "health curious adults" --style "documentary explainer" --source youtube --content-type scriptable --language english

YouTube API setup
Paste the key into Youtube-Agents/.env:
YOUTUBE_API_KEY=your_key_here

Query template design
ResearchAgent uses neutral evidence templates:
- {topic}
- {topic} explained
- {topic} benefits
- {topic} risks
- {topic} mistakes
- {topic} myths
- {topic} what happens
- {topic} science
- {topic} beginner guide

It also adds topic-sensitive evidence templates for health, money, and tech topics, plus optional audience/style/manual context queries. It must not force niche-lane searches like "POV life as {topic}".

Content type
ResearchAgent can filter YouTube evidence by duration:
- both: keep all durations
- shortform: keep videos up to 60 seconds
- midform: keep videos from 61 to 239 seconds
- longform: keep videos at least 240 seconds
- scriptable: keep videos at least 180 seconds

Language
ResearchAgent can filter YouTube evidence with:
- auto: no language filter
- english: reject titles that appear mostly non-English/non-Latin

StrategyAgent handoff
StrategyAgent will later read research.json, choose a recommended niche lane and angle, explain rationale, and produce strategy.json.

StrategyAgent responsibility

StrategyAgent scoring signals
- query matches from competitor videos
- strong_fit_competitors when available

StrategyAgent handoff
IdeaAgent will later read strategy.json and generate candidate video ideas. StrategyAgent must not decide the final title, full outline, script, or thumbnail package.

Competitor fit rule
ResearchAgent should score and bucket competitor fit rather than hard-delete weird outliers. Strong-fit videos should drive strategy. Adjacent opportunities and off-context outliers should remain available for creative inspiration.

IdeaAgent responsibility
Read research.json and strategy.json, generate scored candidate video ideas, select the highest-scoring idea, and produce ideas.json.

IdeaAgent inputs
- research.json
- strategy.json
- ideas_per_run
- max_candidates

IdeaAgent outputs
- ideas.json
- strategy_used
- ideas
- selected_idea
- handoff_contract

IdeaAgent CLI
python3 main.py ideas --research workspace/agent_runs/YYYYMMDD_HHMMSS_topic/research.json --strategy workspace/agent_runs/YYYYMMDD_HHMMSS_topic/strategy.json --ideas-per-run 5 --provider template

IdeaAgent LLM mode
python3 main.py ideas --research workspace/agent_runs/YYYYMMDD_HHMMSS_topic/research.json --strategy workspace/agent_runs/YYYYMMDD_HHMMSS_topic/strategy.json --ideas-per-run 5 --provider openai --model gpt-4o-mini
python3 main.py ideas --research workspace/agent_runs/YYYYMMDD_HHMMSS_topic/research.json --strategy workspace/agent_runs/YYYYMMDD_HHMMSS_topic/strategy.json --ideas-per-run 3 --max-candidates 5 --provider ollama --model llama3.1

IdeaAgent Ollama model defaults
Use llama3.1 for stronger local idea generation. The Ollama client does not cap context size, output length, or keep-alive settings; Ollama uses the model's own defaults.

IdeaAgent provider rule
The LLM only generates raw candidates. IdeaAgent still normalizes, scores, sorts, and writes the same ideas.json contract. If the LLM call fails or returns invalid JSON, it falls back to templates and records the error in generation.

IdeaAgent handoff
OutlineAgent will later read ideas.json and turn selected_idea into a structured outline. IdeaAgent must not decide the full outline, script, thumbnail package, or final edited title.
